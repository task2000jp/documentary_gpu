# v3.1（blessing_documentary）レビューから得た設計教訓

前作の実視聴レビュー（review.md）から抽出した、**次世代パイプラインに最初から組み込むべき原則**。
個別の修正内容は `../../blessing_documentary/docs/revision_v32.md` 参照。

## 映像・テキスト
- **テキストを揺らさない・ポップさせない**。スケール拡大/叩きつけ(punch/slam)は安っぽい。
  既定は静かなフェード(dissolve)。厳粛さ＝動きの少なさ。
  → compositor の text オーバーレイはフェードイン/アウトのみを既定にする。
- **テキストは必ず max_width で自動改行**。画面外はみ出しを構造的に防ぐ。
  特に左右分割レイアウトは中央強調テキストが溢れやすい。安全マージンを設計に。
- **多要素の順次表示は尺を保証**。N要素×間隔がシーン尺を超えると後半要素が出ない。
  scene_schema 側で「要素数×最小表示時間 ≤ duration」を検証する。
- **章の繋ぎ目（暗転）にも短いテキスト**を添えて区切りと余韻を出す。

## 音響（最重要の地雷）
- **全ナレーションをラウドネス正規化**（例 -16 LUFS）してから合成。
  v3.1 は章ごとに音量がバラついた（第5章が小さい）。narration.py に正規化を1パス入れる。
- **SEとナレーションの時刻衝突を避ける**。SEはナレ間隙に配置するロジックを compositor に。
- **効果音は意味に合わせる**。「砕く＝破壊」でなく「砕く＝勝利の打撃」なら、
  ガラス音(キラン)でなく低域インパクト＋ブラス的な勝利音に。

## 内容・脚本（汎用の教訓）
- **専門概念は使う前に説明**。結論（例「○○が原因」）を、根拠の提示前に出さない。
  → [[feedback-simple-language]] と同根。順序＝理解。
- **論理の鎖を1つずつ可視化**。飛躍を作らない（断絶→回復、生産革命→繁栄、など因果を明示）。
- **複数事件を混ぜない**。1シーン1論点。
- **現代への接続**で見応えが増す（産業革命→IT→AI のように射程を伸ばす）。

## 神学的・主張の防御（コンテンツがミッション性を持つ場合）
- **反論を先回りして作中で無効化**する。本作なら「繁栄の神学ではない」を明示。
  主張＝勝利者キリスト(恐怖からの解放)を軸にし、富は"結果・手段"と位置づける。
- **誇示・勝利主義を避ける謙抑的な因果**で語る（「Aが結果としてBを生んだ」）。

## パイプライン運用（今回の手戻りから）
- アセット（画像・BGM）は最初から **Drive を Single Source of Truth** に。
  Colab セッションごとの再アップロードは手戻りの温床。
- `_starts_of` 系の未定義・コピペバグ（lbl/e_lbl）が起きた。
  → シーン定義をデータ(JSON)化し、描画ロジックを1本化すれば構造的に減る（本プロジェクトの設計方針）。
