## 0. 前提：この作品は「バズ」狙いではない

`勝利の福音` は知的・厳粛な神学ドキュメンタリー。マーケ戦略は娯楽チャンネルと根本的に異なる。

| 娯楽チャンネル | このチャンネル |
|---|---|
| トレンド追従・即時性 | **エバーグリーン**（数年効き続ける） |
| サムネ釣り・感情扇動 | **知的好奇心の問い**（釣りは厳禁・トーンを裏切らない） |
| 量産・回転 | **権威の蓄積**（"なぜ"に最も深く答える場所になる） |
| 広く浅く | **唯一の統合**（信仰 × 経済史 × 技術 — 誰もやっていない） |

**このチャンネルの最強の武器** = 作品自体が持つ「問いの構造」。
`なぜ蒸気機関はスコットランドで生まれたか` は完璧なYouTubeタイトル（好奇心ギャップ＋具体＋知的）。
各章も同じ型で独立した問いになる：
- 「なぜギリシャは蒸気機関を"玩具"で終わらせたのか」（第5章）
- 「なぜ豊かだった中国文明は止まったのか」（第5章）
- 「ルターは本当は何を"発見"したのか」（第6章）

## 1. 配信戦略：2階層 + プレイリスト

| 階層 | 形態 | 役割 | 制作 |
|---|---|---|---|
| **本編** | 25分フル + 章別(10本) | 権威・視聴時間・回遊の目的地 | 既存パイプライン |
| **Shorts** | 60秒の問いフック(縦) | 発見エンジン・本編への導線 | **同じパララックスを縦で1シーン描くだけ** |
| **プレイリスト** | 章を順序化 | "講座"化・セッション時間・回遊 | YouTube側設定 |

**重要な気づき**: Shortsは追加コストほぼゼロ。同じ `scene JSON` を `1080x1920` で1〜2シーン描けば縦動画になる。
発見の9割はShorts/検索から来るので、これが無料成長の心臓部。

新規チャンネルの定石 = **検索ファースト → 提案（suggested）へ**。
安定した検索意図（「宗教改革 わかりやすい」「産業革命 なぜイギリス」）を章別動画で押さえ、
権威がたまるとYouTubeが関連動画として提案し始める。

## 2. システム構成：閉ループ（3ループ + 1頭脳）

```
        ┌────────────────────────────────────────────────┐
        │              Claude（戦略頭脳・本部）             │
        │  groq-compound : 競合/需要のリアルタイム調査       │
        │  youtube MCP   : 統計・競合・トレンド取得          │
        │  Groq llama    : 大量メタデータ生成（スクリプト経由）│
        └──────┬──────────────────────────────┬───────────┘
               │                              │
   ┌───────────▼──────────┐       ┌──────────▼────────────┐
   │ Loop A: 需要探索      │       │ Loop C: 効果測定        │
   │ (企画前・Claude直)    │       │ (投稿後・Claude直)      │
   │ キーワード×競合×機会    │       │ youtube MCP → 統計分析   │
   └───────────┬──────────┘       └──────────┬────────────┘
               │                              │
               │     ┌────────────────────┐  │
               └────▶│   Sheets DB         │◀─┘
                     │   (共有メモリ)        │
                     │ ・キーワード戦略       │
                     │ ・投稿管理           │
                     │ ・実績ログ           │
                     └─────────┬──────────┘
                               │ 戦略を渡す
                ┌──────────────▼───────────────┐
                │ Loop B: メタデータ生成         │
                │ (スクリプト・Groq API)         │
                │ scene JSON + 戦略 → Groq      │
                │ → title/desc/tags/chapters    │
                │ → scenes/<ch>_meta.json       │
                └──────────────┬───────────────┘
                               │
                      upload_youtube.py
                               │
                          YouTube 投稿
                               │
                          （実績が Loop C へ）
```

**閉ループ**: Loop C の学び → Loop A のスコア更新 → Loop B の生成改善。
Sheets DB が全ループの共有メモリ。作品をまたいで学習が蓄積する（=スケール設計）。

## 4. 各ループの運用

### Loop A — 需要探索（Claude が随時実行）
1. groq-compound で competitor/トレンド調査（英語クエリ・413回避）
2. youtube MCP `searchVideos` で対象語の上位動画・再生数・チャンネル規模を取得
3. 機会スコア算出 → Sheets「キーワード戦略」へ append
4. 高スコア語を次の制作テーマ・タイトルに反映

**API予算**: `searchVideos`=100units/回。日10,000units枠 → 1日最大~90検索。乱用しない（Claudeが厳選）。

### Loop B — メタデータ生成（`scripts/gen_metadata.py`）
入力: 章の `scene JSON`（タイミング＋テキスト）＋ 任意のブリーフ＋ 戦略キーワード
出力: `scenes/<ch>_meta.json`（`upload_youtube.py` が直接食える）

自動生成内容:
- **タイトル候補**（A/B・問いフック型・全角28字前後・キーワード前寄せ）
- **説明文**（フック → 要約 → 章タイムスタンプ → 出典 → CTA → ハッシュタグ）
- **タグ**（広域 + ロングテール日本語）
- **章タイムスタンプ**（title スタイルのシーンから累積秒で自動算出）★
- **固定コメント**（エンゲージ誘発の問い）
- **サムネ案**（重ねるテキスト + 使用画像）

★章タイムスタンプは `scene JSON` から完全自動。視聴維持率とSEOに直結する最大の無料施策。

### Loop C — 効果測定（Claude が随時実行）
1. youtube MCP `getVideoDetails` で再生数・高評価・コメント数取得（1unit/動画）
2. `getVideoComments` で感情・要望を抽出（次テーマの種）
3. Sheets「実績ログ」へ時系列で append
4. 過大/過小パフォーマンスを分析 → Loop A の機会スコアへフィードバック

## 5. ミッション整合（最重要の制約）

- **釣りタイトル禁止**: 作品の厳粛なトーンを裏切る誇張・扇動はブランド毀損。問いは「誠実な好奇心」で。
- **差別化＝統合**: 信仰 × 経済史 × 技術 の統合は唯一無二。これを薄めず前面に。
- **福音は資産であり負債ではない**: 神学的中身が独自性の源。一般視聴者にも「知的な問い」として開く。
- 目的は再生数ではなく **正しい魂に良質なコンテンツで福音を届けること**。指標は手段。

---

## 6. スケール設計

- Sheets DB は作品列を持つ → 別ドキュメンタリーも同じループで回る
- `gen_metadata.py` は作品非依存（scene JSON + ブリーフを食うだけ）
- 学習（どの問い型・どの語が効くか）は作品をまたいで蓄積
- チャンネルは「作品群＝シーズン」として成長する
