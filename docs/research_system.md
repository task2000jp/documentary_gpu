# 研究部門 — 論旨補完エンジン（自律的・知識ギャップ駆動）

> 目的: **己の知識不足を疑問化し、調査・分析し、論旨に統合する**循環を絶え間なく回し、
> 作品の論理（`content_design.md`）を洗練・完成させ続ける。
>
> 起点となった実例: 「ブリティッシュロック/ハリウッドはなぜ富を集め成功したのか（成功の必然性）／
> それは宗教改革とどう相関するか」→ "インフラとペイロード論"（content_design v3.3）に結実。

---

## 0. 思想：ギャップ駆動

事実を漫然と集めるのではない。**論旨の穴**を見つけ、それを埋める問いを立て、答える。
良い問いのテンプレ（あなたの実例から抽出）:

> **「[事象X] はなぜ成功し資源を集めたのか（成功の必然性）。
>   それは 福音→心の解放→インフラ→ペイロード の論旨とどう関係・相関するか。」**

---

## 1. ギャップ分類（taxonomy）

| 種別 | 問いの形 | 例 |
|---|---|---|
| **成功の必然性** | なぜその事象は成功し富/影響を集めたか | British Rock, Hollywood, Silicon Valley, 日本経済奇跡, K-pop, 大学制度, 保険・複式簿記 |
| **改革との相関** | それは福音→解放→インフラ論とどう接続するか | プロテスタント倫理と資本/技術の連鎖 |
| **未証明前提** | content_designが断定するが論証が薄い箇所 | 「心の解放が先、外が後」の因果順序の根拠 |
| **反証・対抗仮説** | 論旨を弱める事実はないか（誠実さ＝一般恩恵）| 非プロテスタント圏の産業化、世俗的説明 |

---

## 2. 3層アーキテクチャ（コストと深さを対応）

```
L1 自律収集 [GAS + Groq・毎日・無料・Claudeトークン0]
   論旨ギャップの問いを生成 + 一次調査 → Sheets「論旨ギャップ」キュー
        │ 絶え間なく溜まる
        ▼
L2 深堀り統合 [Claude司令・随時・中コスト]
   キューから高優先を選抜 → WebSearch/WebFetch で精査
   → 洞察に合成 → content_design整合チェック → 統合提案
        │ 大型ギャップは ▼
L3 徹底研究 [deep-research skill・重・稀]
   多源fan-out + 反証検証 + 引用レポート → content_design 新節へ
```

**設計判断**: 「自律・絶え間ない」部分は **GAS+Groqに置く**（無料・Claude起動トークンを食わない）。
**知性が要る深い統合**は **Claudeが随時**（トークンを意識し、まとめて実施）。
groq-compound MCPは使わない（413多発・非効率／メモリ参照）。最新性は直接WebSearch。

| 更新日 | |

---

## 4. 統合プロトコル（品質ゲート）

洞察を content_design に入れる前に必ず:
1. **出典**を添える（一次/二次）
2. **対抗仮説**を1つ検討（世俗的説明で十分説明できないか）= 一般恩恵的誠実さ
3. **どの章/節に効くか**を明示
4. **神学的整合**チェック（繁栄の神学と混同しない／論旨鎖を壊さない）

→ 通過したものだけ `content_design.md` に節として統合（v番号を上げる）。

## 5. コンポーネント

| 部品 | 役割 | 状態 |
|---|---|---|
| `scripts/research_engine.py` | Groqでギャップ問い+初期仮説を生成（L1中核）| 実装 |

| `deep-research` skill | L3 大型ギャップの徹底研究 | 運用 |
| GAS thesis-gap ループ | L1を毎日自律実行（次段）| 設計 |

---

## 6. 運用サイクル

1. （自律）L1がギャップ問いを毎日キューに足す
2. （随時）Claudeがキューを見て高優先を数件 L2 深堀り → 洞察
3. 品質ゲート通過 → content_design に統合（vアップ）
4. 大型は L3 deep-research へ
5. 統合済みは状態 integrated。論旨が一段完成 → 新たな穴が見える → 1へ

→ **作品の論理が自己増殖的に洗練されていく。**
