# 研究部門 — 論旨補完エンジン（自律・偏りなき・被造設計駆動）

> 目的: **己の知識ギャップを問い化→調査→分析→論旨統合**の循環を絶え間なく回し、
> 作品の論理（`content_design.md`）と世界理解を洗練・完成させ続ける。

---

## 0. 思想：偏らせない／全ては繋がる

二つの原理が同時に働く。

**(A) 調査対象を偏らせない。**
エリート・政治経済の成功（例: nvidiaの必然性、産業革命）は既に論理展開できる＝**冗長**。
本当のフロンティアは **大衆・日常・関係性** ──「なぜこのアニメ/音楽が当たるか」「なぜあの人は
モテて幸福か」。**成功は富だけではない**。トレンドとランダムで対象を散らし、偏りを殺す。

**(B) しかし全ては繋がる。** 神＝クリエイターが人間・脳・被造世界を作ったから。
だから engine の仕事は連関を**捏造**することではなく、**設計上すでに在る連関を発見**すること。
→ 知的誠実さ（対抗仮説の検討）と、連関への確信（一なる創造者）を**両立**させる。

---

## 1. 成功の多次元（シャローム5要素で偏りを正す）

`content_design.md` のシャローム定義がそのまま「成功の分類」になる。
従来engineは第1次元（外の豊かさ）だけ見ていた＝偏り。全次元を等しく扱う。

| 成功の次元 | シャローム要素 | 例（調査対象）|
|---|---|---|
| 外的 | 外の豊かさ | nvidia, 産業革命, ハリウッド（※エリートは低優先）|
| **関係的** | 関係 | なぜあの人は愛され人望がある／推し・配信者・アイドル |
| **身体的** | 健康 | なぜ美/フィジカルが魅力になる／健康法の流行 |
| **内面的** | 心の平和 | なぜこの音楽/アニメは心に刺さる／幸福な人の共通点 |
| 霊的 | 神との和解 | なぜ人は意味・赦し・超越を求める |

太字＝これまで取りこぼしていた領域。ここを厚く。

---

## 2. 分析の二軸（マクロ＋ミクロ＝被造設計）

各事象を**二つの相関**で問う。これが「全ては繋がる」の実装。

| 軸 | 問い | 適用 | 担当 |
|---|---|---|---|
| **マクロ相関** | 歴史・インフラ・摂理（福音→改革→産業→メディア→ペイロード）とどう繋がるか | 歴史/経済/技術事象 | deep-research / WebSearch |
| **ミクロ相関（被造設計）** | なぜ人間の脳・心は**それに惹かれるよう作られている**か（美/物語/愛着/地位/超越への配線）| **あらゆる事象**（特に大衆・関係）| **research-specialist**（脳科学）|

統合: 両軸は**一なる創造者**に遡る ── 摂理（歴史をこう進めた）と 設計（人間をこう作った）。
だから王朝の興亡も、ヒット曲も、モテる理由も、同じ源に繋がる。

## 3. 偏りなきサンプリング（対象選定）

`--mode` で対象の取り方を変え、混合で偏りを殺す。

| mode | 取り方 | 例 |
|---|---|---|
| trend | 今ホットなもの | YouTube急上昇/チャート/話題 |
| everyday | ユーザーの日常消費 | 視聴アニメ・聴く音楽・ゲーム |
| relational | 関係性の成功 | なぜ人は愛される/モテる/人望 |
| random | 領域横断のサイコロ | 偶然の事象 |
| elite | 政治経済（低優先・既知）| nvidia等 |
| **mixed** | 上記を均衡配合（既定）| 偏り防止 |

各ギャップに `success_type` と `perspective` を付与し、キューが特定次元に偏らないよう監視。

## 4. 自動化アーキテクチャ（Claudeを端へ・GASが回す）

**原則: 日々の研究・分析・自己深化は全自動（GAS+Groq・無料・Claudeトークン0）。**
Claudeは「自動化を作る／稀に監査・昇格する」端役に退く。手動の一回限り研究はしない。

```
[毎日・全自動 GAS+Groq70b] gas/research.gs runResearchAll()
  生成    generateGaps_    : 偏りなき二軸ギャップ生成 → 論旨ギャップ(open)
                            ↑C 研究改善ログの最新点検を読んで生成を是正（自己改善ループ）
  A調査合成 researchOpenGaps_: open を【多源】調査(日Wikipedia+英Wikipedia+arxiv認知科学)
                            +Groq合成 → 洞察(resolved)。被造設計軸をarxivで裏づけ
  B昇格起案 draftPromotions_: resolved → content_design草案を自動起案＋品質ゲート自己適用
                            (出典/対抗仮説/神学整合) → 昇格ドラフト(待承認), gap→proposed
  C自己深化 selfDeepen_     : 週1 キュー偏り点検→次の攻め方 → 研究改善ログ → 生成へ還流
        ▼ 知識がSheetsに自己増殖し、システム自体も自己改善（人もClaudeも不在で）

[稀・人/Claude] 昇格承認のみ（高コストゆえ最小化）
  昇格ドラフトを確認 → 良ければ content_design.md に貼る(vアップ)。起案は自動、承認だけ人。
  ＋ 自動化自体の改善（このシステムを作る/直す）= 我々の会話の本来の目的

[大型のみ] deep-research skill / research-specialist(脳科学) ← ユーザー指示時
```

groq-compoundは使わない（413多発／メモリ参照）。研究用GroqはcallGroq_(llama-3.3-70b・JSON強制)。
**自己改善ループの閉鎖**: selfDeepen_→研究改善ログ→generateGaps_が翌日読む→生成が是正される。

### 自動化と人間ゲートの境界
GASは**知識ベース(Sheets)を自律的に積み上げる**。canonical な `content_design.md`(git)への統合だけは
品質ゲート（出典/対抗仮説/神学整合）を通す**昇格**で、Claude/人が承認。
→ 昇格用ドラフトもGroqが自動起案（研究改善ログ）。人は承認するだけ。

## 5. データモデル：Sheets 3タブ

**論旨ギャップ**（メインキュー・16列）:
ID / 領域 / 事象 / **成功次元** / **視点** / 問い(機序) / 問い(マクロ相関) / **問い(被造設計)** /
初期仮説 / 対抗仮説 / 優先度 / 状態 / 洞察 / 統合先 / 出典 / 更新日
状態遷移: open → researching → resolved → proposed →(承認)→ integrated

**昇格ドラフト**（B出力・承認待ち）:
日付 / gap_id / 事象 / 統合先 / content_design草案(md) / 出典 / 対抗仮説チェック / 神学整合 / 品質ゲート / 承認状態

**研究改善ログ**（C出力・生成へ還流）:
日付 / 偏りの所見 / 次に厚くする領域 / 新しい問いの型 / 成功次元分布 / 視点分布

---

## 6. 統合プロトコル（品質ゲート）

統合前に必ず: ①出典 ②対抗仮説（世俗的説明で足りないか）③どの章/節に効くか
④神学的整合（繁栄の神学と混同しない・シャロームの射程）。通過分のみ content_design に統合(vアップ)。

## 8. 運用サイクル

1.（自律）L1が偏りなき二軸の問いを毎日キューへ
2.（随時）Claudeが高優先を選抜 → マクロ(WebSearch)＋ミクロ(脳科学)で深堀り → 洞察
3. 品質ゲート → content_design 統合（vアップ）
4. 大型は L3
5. 一段完成 → 新たな穴 → 1へ。**論理が自己増殖的に洗練。**
